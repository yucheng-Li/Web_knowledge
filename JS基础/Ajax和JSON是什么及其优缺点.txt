Ajax : Ajax不是JavaScript的规范，它是一个 哥们的’发明‘ ：Asynchronous JavaScript and XML 意思是用JavaSript执行异步网络请求
	就想我们之前用form来提交表单一样，每按一次submit按钮我们的网页都会停下来去等待资源请求返回，而这也是web的请求原则
	一个请求对应一个页面，因此如果我们想要发送请求同时保留当前的操作页面，我们就必须要用JavaScript来发送新的请求，收到响应后
	我们可以通过回调函数来获取响应
	
Ajax的优点：
	（1）发送请求即响应时无页面刷新，用户体验好
	（2）使用异步和服务器通信，具有更快的响应速度（两个五秒的请求，如果是同步就是10秒，如果是异步就是5秒多一点）
	（3）Ajax的原则按需请求，把以前一些服务端的工作转嫁到客户端，减轻了服务器和带宽的压力
	（4）Ajax基于标准化并且得到了广泛的技术支持，无需下载其它的插件

Ajax的缺点：
	（1）Ajax 取消了浏览器的back和history的功能，即浏览器的机制被破坏；
	（2）安全问题，Ajax暴露了客户端与服务器交互的细节；
	（3）对搜索引擎支持较弱	
	（4）破坏程序的异常机制
	（5）违背了URL和资源定位的初衷
	
	
JSON：JSON是JavaSript Object Notation 的缩写，它是一种数据交换格式，JSON出现之前大家都是用XML的传递数据的
	但后来由于出现了一大堆XML规范后，XML就变得十分复杂，而后JSON就应运而生并以轻量简便著称，JSON实际上是
	JavaScript的一个子集一共有number\boolearn\string\null\array\object这几种数据类型，以及它们的组合，且JSON规定
	死了字符集必须使用UTF-8来解决多语言问题
	
JSON的优点：
	（1）数据格式比较简单，易于读写，格式都是压缩的，占用带宽小
	（2）易于解析
	（3）支持多种服务端语言
	（4）JSON格式能直接为服务器端代码使用，易于维护
	
JSON缺点：
	（1）没有XML推广那么好深入人心，没有XML通用性强
	（2）JSON格式目前在Web Service中推广还属于初级阶段
